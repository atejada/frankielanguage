    Frankie Web Server

    Frankie ships a built-in HTTP web server in the style of Sinatra / Camping. Zero external dependencies — it runs on Python's standard library http.server.

    Quick Start

    app = web_app()

app.get("/") do |req|
  html_response("<h1>Hello from Frankie! 🧟</h1>")
end

app.run(3000)

    frankiec run myapp.fk
# 🧟 Frankie web server running on http://0.0.0.0:3000

    Routes

    Register routes with .get, .post, .put, .delete, .patch. Each takes a path pattern and a do |req| ... end block. The block receives a FrankieRequest and must return a response value.

    app.get("/hello") do |req|
  response("Hello!")
end

app.post("/echo") do |req|
  response(req.body)
end

app.delete("/items/:id") do |req|
  response("Deleted #{req.params["id"]}")
end

    Path Parameters

    Use :name segments to capture parts of the URL:

    app.get("/users/:id/posts/:slug") do |req|
  id   = req.params["id"]
  slug = req.params["slug"]
  response("User #{id}, post #{slug}")
end

    Query Parameters

    # GET /search?q=frankie&page=2
app.get("/search") do |req|
  q    = req.query["q"]
  page = req.query["page"]
  response("q=#{q} page=#{page}")
end

    Request Object

    Property Type Description
    method String "GET", "POST", "PUT", "DELETE", "PATCH"
    path String URL path, e.g. "/users/42"
    params Hash Path parameters, e.g. {id: "42"}
    query Hash Query-string parameters, e.g. {page: "2"}
    headers Hash HTTP request headers
    body String Raw request body
    json Hash Body parsed as JSON, or nil if not valid JSON
    form Hash Body parsed as application/x-www-form-urlencoded
    cookies() Hash Parsed Cookie header — {name: value} pairs

    Response Helpers

    Function Status Content-Type
    response(body) 200 text/plain
    response(body, status) custom text/plain
    response(body, status, headers) custom text/plain
    html_response(html) 200 text/html
    html_response(html, status) custom text/html
    json_response(hash) 200 application/json
    json_response(hash, status) custom application/json
    redirect(location) 302 —
    redirect(location, status) custom —
    halt(status, body) custom text/plain
    render(path, data) 200 text/html

    Returning a plain string from the handler is also fine — it becomes a 200 text/plain response automatically. Returning a hash or vector auto-converts to JSON.

    Response Methods

    resp.set_cookie(name, value, opts) — append a Set-Cookie header. opts is an optional hash:

    Key Default Description
    path "/" Cookie path scope
    max_age omitted Lifetime in seconds
    same_site "Lax" "Strict", "Lax", or "None"
    http_only true Set HttpOnly flag

    resp.set_header(name, value) — set an arbitrary response header.

    Both methods return the response object so they can be chained.

    Cookies

    Reading cookies

    app.get("/dashboard") do |req|
  session = req.cookies["session"]
  if session == nil
    redirect("/login")
  else
    html_response("<p>Welcome back!</p>")
  end
end

    Writing cookies

    app.post("/login") do |req|
  token = "abc123"
  resp  = html_response("<p>Logged in</p>")
  resp.set_cookie("session", token, {max_age: 86400, same_site: "Strict"})
  resp
end

    Clearing a cookie

    Set max_age to 0:

    app.get("/logout") do |req|
  resp = redirect("/")
  resp.set_cookie("session", "", {max_age: 0})
  resp
end

    Before / After Filters

    Logging (after only)

    app.after do |req, res|
  puts "#{req.method} #{req.path} -> #{res.status}"
end

    Short-circuiting with halt

    A before-filter can stop the request entirely by calling halt. The route handler is never invoked.

    app.before do |req|
  token = req.cookies["session"]
  if token == nil
    halt(401, "Unauthorized")
  end
end

app.get("/protected") do |req|
  response("secret data")
end

    Any halt inside a before-filter immediately returns its response to the client and skips all remaining before-filters, the route handler, and all after-filters.

    after filters still run normally when the request completes through the route handler.

    Static File Serving

    app.static(url_prefix, fs_root) maps a URL prefix to a directory on disk.

    app = web_app()
app.static("/assets", "./public")

# GET /assets/style.css  ->  ./public/style.css
# GET /assets/js/app.js  ->  ./public/js/app.js

app.run(3000)
    • MIME types are inferred from the file extension (text/css, application/javascript, etc.)
    • Missing files return 404
    • Path traversal attempts (e.g. /../etc/passwd) return 403
    • Static routes are checked before dynamic routes

    Template Rendering

    render(template_path, data) reads a .html file from disk, fills {{key}} placeholders using the template() function, and returns an html_response.

    # views/index.html
# <h1>{{title}}</h1>
# <p>Hello, {{name}}!</p>

app.get("/") do |req|
  render("views/index.html", {title: "Home", name: "Alice"})
end

    The placeholder syntax is the same {{key}} used by the standalone template() function. Keys not present in data raise a runtime error.

    JSON API

    # Receive JSON
app.post("/api/users") do |req|
  data = req.json
  if data == nil
    halt(400, "Expected JSON")
  else
    name = data["name"]
    json_response({id: 1, name: name}, 201)
  end
end

# Send JSON
app.get("/api/info") do |req|
  json_response({version: "1.14", alive: true})
end

    Custom 404

    app.not_found do |req|
  html_response("<h1>404</h1><p>Nothing at #{req.path}</p>", 404)
end

    Starting the Server

    app.run(3000)               # listen on 0.0.0.0:3000 (default)
app.run(8080)               # custom port
app.run(3000, "127.0.0.1")  # localhost only

    The server is multi-threaded (one thread per request) and blocks until Ctrl+C.

    Web API Summary

    Function / Method Description
    web_app() Create a new application
    app.get(path) do \|req\| end Register a GET route
    app.post(path) do \|req\| end Register a POST route
    app.put(path) do \|req\| end Register a PUT route
    app.delete(path) do \|req\| end Register a DELETE route
    app.patch(path) do \|req\| end Register a PATCH route
    app.before do \|req\| end Before-filter — call halt to short-circuit
    app.after do \|req, res\| end After-filter
    app.not_found do \|req\| end Custom 404 handler
    app.static(prefix, dir) Serve files from dir under prefix
    app.run(port) Start the server (blocking)
    response(body, status, headers) Plain-text response
    html_response(body, status) HTML response
    json_response(data, status) JSON response
    redirect(location, status) Redirect response
    halt(status, body) Short-circuit — works from before-filters too
    render(path, data) Render a .html template with {{key}} values
    resp.set_cookie(name, val, opts) Append Set-Cookie header
    resp.set_header(name, val) Set arbitrary response header
    req.cookies() Parsed Cookie header as a hash
